Anatomy of CSS

Structure

  1. Selector: specify what part of the structure you want to target
    • Types of Selectors:
      • Element: just specify the element name (e.g. p)
      • Class: specified with a . in front of the class name
      • ID: specified with a # in front of the id name
      • * (star selector): specifies every single element
    • Combining Selectors:
      • Selector1, Selector2: OR Selector - Both Selector1 and Selector2 elements will be targeted
      • Selector1Selector2: AND Selector - Targets the element that fits both criterion
        • E.g. p.big selects only p elements with the class big
      • Selector1 > Selector2: Child selector - Targets every Selector2 element that is a direct child of a Selector1 element
        • E.g. div > p targets all p elements that is a direct child of a div element
      • Selector1 Selector2: Descendant Selector - Targets every Selector2 element that is a descendant of Selector1
        • E.g. div p targets all p elements that is a descendant of a div even if it's not a direct child of it
      • Selector1 + Selector2: Adjacent Sibling Selector - Targets ONLY the Selector2 that IMMEDIATELY FOLLOWS Selector1 element and both have the same parent
      • Selector1 ~ Selector2: General Sibling Selector - Targets ALL Selector2 elements that follow the first element (not necessarily immediately) and have the same parent
    • Pseudo-Class Selectors
      • Usage: selector:pseudo-class
        • :link - this pseudo-class targets elements with a link
        • :visited - this pseudo-class targets elements with a clicked link
          • Use text-decoration: none; to remove the underline on the link
        • :hover - this pseudo-class targets elements that are being hovered over by mouse
        • :active - this pseudo-class targets elements that are being clicked on but the user hasn't released his click
        • :nth-child() - this pseudo-class targets the specific nth child of an element (number inside brackets) (can also target all odd/even elements using odd/even instead of number in brackets)
      • Note that you can stack pseudo-class selectors by doing selector:pseudo-class1:pseudo-class2
        this targets the element when both pseudo-class requirements are met (AND)
  2. Open bracket: {
  3. Declaration: consists of 2 parts - property and value
    • There are a predefined set of properties and a set number of predefined values for each property
    • It's good practice to end a declaration with a semi-colon
  4. Close bracket: }
