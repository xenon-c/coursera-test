CSS Styling

Notes

  • Style Placement
    • Inline Styling: done as attribute of an element (no need to specify target), can put multiple styles, just end with ; after each. (Great for testing)
    • External Stylesheets: an external stylesheet file, (file type: .css), linked in head with link element, attribute href="./style.css" (Good for real sites)
  • Conflict Resolution
    • Origin Precedence: LAST DECLARATION WINS (HTML is processed from top to bottom) (So if there's stylesheet linked in head, think of style as placed in head, so any styling below wins)
    • Inheritance: By targeting a parent element, every descendant of that element will inherit that property
    • Specificity: Most specific selector combination wins (style="" > ID > Class/Pseudo-Class/Attribute > # of Elements used)
    • Overwriting: !important after a declaration overwrites everything (it is top priority and cannot be overwritten) (AVOID as it will cause a mess)
  • Font Styling
    • font-family: put a few fonts as when you specify font family, you're relying on the user's computer to have the font installed (as a backup) (specify serif vs sans serif as final measure)
    • color: #RRGGBB (hexadecimal value: first 2 numbers red value, middle 2 green, last 2 blue)
    • font-style: italic
    • font-weight: bold or a numerical value (bold = 900)
    • font-size: give it a numerical value in pixels (e.g. 24px)
    • text-transform: lowercase or uppercase
    • text-align: align right, left, or center
  • Font Size
    • %: take the default size and increase it by n% (E.g. 120% = increase default font size by 120%)
    • em: 1 em = width of the ltter m in the particular font we're using (relative)
    • Note: Font size has a cumulative effect instead of overwriting effect when you specify them in different order (E.g. 1em + 2em = 3em)
    • Note: Relative font size is good if user changes zoom/browser size (as font size will still be relative)
