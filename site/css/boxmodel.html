The Box Model

Notes

  • In HTML, every element is considered a box: They have a padding (around content), border (around padding) and margin (around border)
  • The padding and margin have a default value by the user's agent stylesheet
  • You can set values for each by specifying it in css (E.g. padding: 10px, border: 5px solid black, margin: 40px)
    • If you set a width for a "box", the actual box width is the number set plus total padding, and border width
    • The width/height set is only set for the content by default, and doesn't include padding and border
    • If you change the box-sizing property to border-box, the width and height set = the width and height of box (default: content-box)
  • Note that box-sizing property is not inherited: you can't set it on a parent element expecting child to inherit it
    • You can use the star selector to make it so every element has this property set to border-box
  • Note that margin and padding properties are inherited: if body element is set to margin and padding 0, every element inside of it will be the same (unless specified otherwise)
  • Cumulative Margins
    • If a content box has margin of 40px, and another content box next to it has margin of 50px, cumulative margin will be 90px
    • If a content box has a margin of 20px and another box below it has a margin of 30px, cumulative margin will be 30px
      • The larger margin will win and the other margin will collapse
  • Overflow:
    • If you set a height for a box that cannot contain all the content, content will spill over to the next box and create a mess (as overflow is set to visible by default)
    • Hidden: content will be cut short where box ends
    • Auto: puts scroll bars where needed so you can scroll through content
    • Scroll: puts scroll bars both ways even if uneeded (e.g. horizontal scroll + vertical scroll when you only need vertical)
    • Note that scroll bars are useful for Terms of Service agreements where it's expected to all be packed in a big chunk
