Responsive Design

Notes

  • Responsive Website: a site designed to adapt its layout to viewing environment by using fluid, propertion-based grids, flexible images and CSS media queries
  • Site's layout adapts to the size of the device - Content verbosity or its visual delivery may change
  • Alternative to Responsive Design: Have server detect user agent (mobile vs desktop - mobile = m.website.com, desktop: www.website.com)
    • Mobile devices are too varied in sizes themselves so its hard to make a site that satisfies all mobile clients
  • Phones automatically zoom out unless told otherwise to try to fit the entire contents of the website on the screen
    • Tell the phone's browser to stay at regular zoom level by specifying a meta tag
    • Use <meta name="viewport" content="width=device-width, initial-scale=1">
    • this makes the browser consider the device's width as the browser width and scale of viewing as 1 (won't scale up or down)
  • Use % to achieve fluid width

12 Column Grid Responsive Layout

  • Factors of 12: 1, 2, 3, 4, 6, 12 - You can sub-divide site into sections that are evenly split and nicely layout themselves
  • Browser width = 100%, so 1 column = 100%/12 = 8.33%
  • You can even have nested grids - another 12 column grid layout inside of the sections
