JavaScript Types

Notes

  • A type is a particular data structure
    • Each language defines some built in types
    • Built-in types can be used to build other data structures
    • JS has 7 built-in types: 6 primitive and 1 Object type

Object Type

  • Object is a collection of name/value pairs
    • E.g. firstName (name): "Joe" (value),
      lastName: "Mama",
  • Objects can be nested in other objects
    • E.g. social (name): {
      twitter: "@joemama",
      instagram: "@joemama"} (value)

Primitive Types

  • Primitive type represents a single (single value, i.e. NOT object), immutable (immutable: once value is set, it can't be changed - read-only value) value
  • Boolean: Can only have 2 values - true or false
  • Undefined: signifies no value has ever been set on this variable
    • variable value is set to undefined by default if you didn't give it a value after creating it
    • You can set a variable to undefined, but you should NEVER do it, because its meaning is that its never been defined, so defining it and setting it to undefined is counter ot its core meaning
  • Null: signifies lack of value (lack of definition)
    • Can only have 1 value: null
    • It's ok to explicitly set a variable to null
  • Number: the only numeric type in JS
    • Always presented under the hood as a double-precision 64-bit floating point (JS has no integer type - integers are a subset of doubles instead)
  • String: a sequence of characters used to represent text
    • use either single or double quotes
  • Symbol: new to ES6, not covered in this class
