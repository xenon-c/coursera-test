Setup

Basic Commands

  • console.log() - logs the value inside brackets to the console

Variables

  • Defining a Variable: var variableName = variableValue
    • for a string (text), add double quotes on both sides of the value
    • no types are declared (JS automatically figures out the type of the variable when it is run)

Functions

  • Creating a Function: function functionName (arg) {...}
    • an alternative way of doing this is: var functionName = function () {...}
    • both ways allow you to refer to the function as functionName, though note that no function name is declared after the function keyword (in the variable method)
    • arguments (to be given when function is invoked) are defined without var (E.g. function compare (x, y) {...}
  • Invoking Function: functionName(); (executes the function functionName)
  • Return Value from Function: return value;
    • E.g. function compare (x, y) {return x > y;}
    • var a = compare(4, 5); (return value is stored inside a)

Scope

  • Global
    • variables and functions defined here are available everywhere
  • Function (lexical)
    • variables and functions defined here are available only within this function

Scope Chain

  • Everything is executed in an Execution Context
  • Function invocation creates a new Execution Context
  • Each Execution Context has its own Variable Environment (where it stores its newly defined functions/variables), a special 'this' object, and a reference to its outer environment (execution environment in which this function is defined)
  • Global scope has no outer environment as it is the most "outer" there is
  • Referenced variable will be searched for in its current scope (where the function is physically defined) first. If not found, outer reference will be searched. If not found, outer reference's outer reference will be searched, etc. This keeps on going until Global scope. If still not found, variable is undefined
