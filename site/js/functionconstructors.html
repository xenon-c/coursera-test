Function Constructors

The "this" variable

  • the 'this' variable is actually referencing to the global "window" object
  • if you create a property for the this variable, you are creating a property for the global window object
  • when you put the keyword 'new' before invoking a function to be stored in a variable (E.g. var myCircle = new Circle(10);), the this keyword inside the function will point to the new object created (object gets named after the function that created it, so if your function name is Circle, object name will be Circle)
    • This type of function is called a "Function Constructor"
    • Note that you can't return anything from the function or it won't work as desired

Prototype

  • You can create a property containing a function in the function constructor, but it will be created in every instance of the object (wasted processing)
    • If you want the function to be constant regardless of instance, you can do it with "prototype"
    • E.g. Circle.proptotype.getArea = function () { return Math.PI = Math.pow(this.radius, 2);}
    • This creates a single constant function that will be in every single instance of the object created

Object Literals

  • If you create a object as a variable and inside it you have a property which is a function, the 'this' keyword specified in it will refer to the actual object that got created
  • However, when you have a function inside a function, the this keyword starts pointing to the global object "window"
    • You can get around this by creating a variable and setting it's value to "this" in the first function, then refer to that variable in the functions inside and so on
