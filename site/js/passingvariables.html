Passing Variables

Passing Variables by Value

  • Given b = a, passing/copying by value means changing copied value in b does not affect the value stored in a and vice versa
  • Primitives are passed by value
    • When you create a variable, you are creating a memory slot with a value for that variable
    • Then when you override the value of the variable, you are simply changing the value of the variable in the same memory slot
  • *this concept can be applied to functions (as when you pass an argument into a function, what's really happening is the value of the argument is copied to the value of the "argument variable" of the function)

Passing Variables by Reference

  • Given b = a, passing/copying by reference means changing vopied value in b does affect the value stored in a and vice versa
  • Objects are passed by reference
    • When you create an object with a property, you are creating 1 memory slot for the value of each property, and 1 memory slot for the object which the value of it will be the memory slot of the property (referencing the memory slot of the property)
    • So when you create a variable (b) which equals to a, you're actually creating a new memory slot referencing to the same memory slot(s) for the propertie(s)
    • So when you update a property of the variable b, you're actually updating the value of the same memory slot variable a is referencing to, so a is affected as well (a will have the updated property value of b)
  • *this concept can be applied to functions (as when you pass an argument into a function, what's really happening is the value of the argument is copied to the value of the "argument variable" of the function)
