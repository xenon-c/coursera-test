Functions

Notes

  • FUnctions are first-class data types: whatever you can do with a variable/object, you can also do it to function
    • E.g. you can give functions properties just like an object, so if you have a function called multiply, you could create a property for it by doing multiply.version = "v1.0.0";
  • If you want to call the function, put brackets after the function name. If you simply want to output it's value(s), just specify the name and don't add brackets behind
    • The value of a function is simply the code of the function itself

Creating Functions with Functions

  • You can create a function with another function by making a function within a function, then returning the value of the function in the function:
    • function makeMultiplier(multiplier) {
         var myFunc = function (x) {
             return multiplier * x;
         };
         return myFunc;
      };

      var multiplyBy3 = makeMultiplier(3);
      console.log(multiplyBy3(10));
      • var multiplyBy3 = makeMultiplier(3) - a variable called multiplyBy3 is created, and it's value is the code of myFunc (as myFunc is returned my makeMultiplier function). As the value 3 is passed into multiplier, the myFunc returned will contain the code return 3 * x when it is invoked.
      • multiplyBy3(10) - will output 30 as you pass the value 10 into the function multiplyBy3 and so it returns 3 * 10 (x = 10)

Passing Functions as Argument

  • function doOperationOn(x, operation) {
       return operation(x);
    };

    var result = doOperationOn(5, multiplyBy3);
    • doOperationOn(5, multiplyBy3); - brackets are not added to multiplyBy3 because you just want to pass on the value of it, not invoke it right there
    • result will be 15 as doOperationOn(5, multiplyBy3); returns the value of multiplyBy3(5) = 3* 5 = 15
