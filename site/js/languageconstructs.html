Common Language Constructs

Notes

  • variable += " string"; adds "string" to the back of the variable (basically equal to variable = variable + " string";)
  • NaN represents Not a Number
  • you can define 2 variables by doing var x = 4, y = 5;
  • if you want to check if 2 variables are equal, make sure you put 2 equal signs between x and y to siginify its a comparison and not defining the first variable
  • Type coercion: language converts something for you from one type to another automatically (so '4' as a string = 4 as a number)
    • if you don't want this, use triple equals (===), so if they're not the same type, it's not going to try compare them at all
    • null, undefined, "", 0, NaN automatically gets coerced to being false (as a Boolean value)
    • Any unempty string, and any non-zero number (including negative numbers) will be coerced to true (as a Boolean value)
    • if you have an undefined variable that you add to a string, JS will coerce it into "undefined" as a string
  • OR operator is represented by ||
  • AND operator is represented by &&
  • If you put curly braces on the next line after return, function returns nothing, but if you put the starting curly brace on the same line as return, function returns the things in the curly braces

For Loops

  • for (var i = 0; i < 10; i++) {sum = sum + i;}
    • var i = 0: initialization
    • i < 10: Condition checked by JS to see whether loop should be run again
    • i++: the action that should be taken when loop is run (++ = +1)
